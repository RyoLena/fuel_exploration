```bash
# 第四个终端
cd ros2_ws
source install/setup.zsh
ros2 launch fuel_exploration exploration_nodes.launch.py min_goal_distance_m:=0.25 revisit_block_radius_m:=0.35 lookahead_distance:=0.25 max_angular_speed:=1.6
```
参数说明：
**视点生成**
- candidate_clearance_m
    - 候选视点离障碍物至少要多远
    - 越大越安全，但可选视点会变少
- max_view_range_m
    - 视点评分时，最多考虑多远的 frontier cell
    - 相当于你的“传感器最大有效观察距离”
**目标选择**
- min_goal_distance_m
    - 新目标距离机器人至少要有多远
    - 防止刚到一个点，又选一个几乎贴着自己的点
- revisit_block_radius_m
    - 已访问过的视点周围多大范围内，禁止再次选为目标
    - 用来避免两个相邻点来回横跳
- goal_reached_tolerance_m
    - 机器人离当前目标多近时，算“已经到达这个视点”
    - 到达后会把这个点记进 visited
**路径规划**
- path_clearance_m
    - A* 路径离障碍物至少保持多远
    - 越大越保守，越不容易擦墙，但也可能更容易找不到路
**路径跟踪**
- lookahead_distance
    - 跟踪路径时，控制器往前看多远的路径点
    - 小：更贴路径，但容易抖
    - 大：更平滑，但可能拐弯变钝
- goal_tolerance
    - 跟踪器离路径终点多近时，认为“路径完成”
    - 这个是控制器层的到达判据，不是前面选目标层的
**速度**
- max_linear_speed
    - 最大前进速度
- max_angular_speed
    - 最大转动角速度
**调参经验**
- 机器人来回抖：
    - 先调大 lookahead_distance
    - 调小 max_angular_speed
- 老是贴墙：
    - 调大 candidate_clearance_m
    - 调大 path_clearance_m
- 总在两个点之间犹豫：
    - 调大 revisit_block_radius_m
    - 调大 min_goal_distance_m
- 探索不彻底（总有大片未知区域没去）：
    - 调小 candidate_clearance_m（允许在更靠近障碍物的地方采样视点）
    - 调小 path_clearance_m（让 A* 路径更贴近墙壁，覆盖更多区域）

## 一、FUEL 是什么

FUEL 是一个用于未知环境自主探索的算法。它通过将前沿点聚合成簇，利用 TSP 全局排序后按最短路径依次遍历访问，并在探索过程中增量式更新簇及访问顺序，实现高效探索。

核心优势：
1. **全局最优 vs 贪心**：TSP 全局排序替代"每次去最近前沿"的贪心策略，路径更短，比现有方法快 3-8 倍
2. **增量更新 vs 全局重算**：只更新地图变化部分的前沿信息，支持高频重规划
3. **视点预计算**：簇创建时就生成视点和代价，规划时直接查表，减少实时计算

---

## 二、核心概念

### 前沿 (Frontier)

已知自由区域与未知区域的交界栅格。机器人走向前沿，传感器就能照亮未知区域。

```
已知自由区域    前沿     未知区域
  . . . . .  | ■ ■ ■ | ? ? ? ? ?
  . . . . .  | ■ ■ ■ | ? ? ? ? ?
  . . . . .  | ■ ■ ■ | ? ? ? ? ?
```

### 前沿簇 (Frontier Cluster)

相邻的前沿格子聚成一团，代表"一片需要探索的区域"。

### 视点 (Viewpoint)

一个视点 = 位置 + 朝向 (p, ξ)，表示"站在位置 p，面朝方向 ξ，能观测到该簇的若干前沿格子"。

## 三、两大模块

FUEL 由两大模块组成：

```
模块A：前沿信息结构 (FIS) —— 负责数据维护
模块B：分层规划 (Hierarchical Planning) —— 负责决策执行
```

---

## 四、模块A：前沿信息结构 (FIS)

每次地图更新（传感器扫到新区域）时触发。

### 4.1 前沿检测与聚类

1. 在地图变化区域内检测前沿格子（已知自由 且 相邻未知）
2. 采用**区域增长 (region growing)** 将相邻前沿格子连接成簇
3. 簇过大时，用 **PCA** 找主轴方向，沿主轴切成两半
4. 递归切割，直到每个簇大小合适

### 4.2 簇创建时同时做两件事

**（1）生成候选视点**

- 以簇中心为原点，建立柱坐标系（2D 为极坐标），均匀采样候选位置
- 每个采样点选择使传感器覆盖最多前沿格子的朝向
- 排除被障碍物遮挡的格子
- 按覆盖数量降序排列，最多保留 15 个视点

```
              前沿簇
            ■ ■ ■ ■ ■
           /    |     \
          /     |      \
  视点 o    视点 o    视点 o    ← 在自由区域内采样
          \     |      /
           \    |     /
            视点 o
```

**（2）计算簇间连接代价**

- 每对簇之间，用 A* 搜索路径，代价 = max(路径长度/最大速度, 朝向变化/最大角速度)
- 增量更新：新簇只需计算与已有簇的代价，不用全部重算

### 4.3 增量更新机制

- 只在地图变化区域 (B_m) 内做检测
- 用 AABB 包围盒快速筛选受影响的簇（宽相位检测）
- 删除过时前沿簇，添加新前沿簇
- 簇间代价也随之增量更新

## 五、模块B：分层规划

### 5.1 第一层：全局巡访规划（先去哪个区域？）

- 将所有前沿簇视为 TSP（旅行商问题）中的"城市"
- 利用 FIS 中预存的簇间代价构建代价矩阵
- 求解 ATSP（非对称旅行商问题），得到全局最优访问顺序
- 使用 LKH 启发式求解器
- 额外引入运动一致性代价，避免来回折返

```
输入：10 个簇 + 它们之间的代价
输出：簇2 → 簇5 → 簇1 → 簇4 → 簇3 → ...（全局最优顺序）
```

### 5.2 第二层：局部视点优化（具体站哪看？）

- 在全局顺序上，取离当前位置 R_rf 范围内的连续几个簇
- 这些簇各有最多 15 个视点
- 构建有向无环图，用 **Dijkstra** 搜索最优视点组合
- 目的：单独看每个簇选最优视点，不一定是全局最优组合

```
簇A 视点: a1 a2 a3
簇B 视点: b1 b2 b3

单独最优: a1, b1（但 a1→b1 可能绕很远）
组合最优: a3, b2（总代价更小）
```

### 5.3 第三层：轨迹生成（怎么走过去？）

- 基于 B-spline 曲线生成平滑轨迹
- 优化目标：最小化总时间，同时满足安全、动力学约束
- 起点状态平滑衔接当前运动状态

### 5.4 触发时机

**不是到达目标后才重规划，而是地图一更新就立即触发三层规划。**

机器人在移动过程中传感器不断扫到新区域，可能：
- 原来的前沿已经消失（路上被扫到了）
- 出现了更好的目标

实时重规划让机器人始终做当前最优决策。

## 六、完整流程

```
机器人启动，传感器扫描
        │
        ▼
┌─ FIS 数据维护 ──────────────────────┐
│  1. 检测前沿格子                      │
│  2. 区域增长聚类，PCA 切割大簇        │
│  3. 生成视点（最多15个/簇）           │
│  4. 计算簇间连接代价                  │
└──────────────┬──────────────────────┘
               │
               ▼
┌─ 分层规划 ──────────────────────────┐
│  第一层：TSP 全局排序所有簇           │
│  第二层：Dijkstra 优化近处视点组合    │
│  第三层：生成轨迹，执行               │
└──────────────┬──────────────────────┘
               │
               ▼
         机器人移动，传感器扫到新区域
               │
               ▼
         地图更新 → 回到 FIS 数据维护
               │
         （循环直到没有前沿 → 探索完成）
```

## 七、2D 地面机器人迁移方案

| 维度 | FUEL 原版 (3D UAV) | 2D 小车 |
|------|-------------------|---------|
| 地图 | 3D 八叉树 (OctoMap) | 2D 栅格地图 |
| 前沿 | 3D 体素表面 | 2D 栅格边界线 |
| 视点 | (x, y, z, yaw, pitch) | (x, y, θ) |
| 传感器 | 3D 深度相机 | 2D 激光雷达 |
| 轨迹 | B-spline (6-DOF) | Nav2 (DWB/MPPI/TEB) |
| 计算量 | 高 | 显著降低 |

技术栈：ROS2 + Nav2 + slam_toolbox
- 第一、二层：自己实现（探索规划节点）
- 第三层：Nav2 负责（NavigateToPose）
